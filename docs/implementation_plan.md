# 実装方針メモ（相槌のズレを減らす）

この文書は、いま決まった方針を、実装の手順に落とすためのメモです。狙いは「相槌のズレ」と「返しすぎ」を減らし、聞き取りを邪魔しないことです。

## 0. いま何が起きているか（観察）

現状は大きく2つの流れがあります。

1つ目は IMU を常時読み、短い窓の統計から相槌サインらしさを出します。2つ目は、サインが立ったらエージェントを呼び、mp3 を再生します。ここで、サインが立つと「返すこと」が前提になりやすく、エージェントが後から理由を作れてしまいます。結果として、少し動いただけで反応が増えたり、タイミングが悪いと聞き取りを邪魔したりします。

## 1. ゴール（何を正解にするか）

ここでは、返すことよりも、邪魔をしないことを正解にします。迷うなら返さないを基本にします。相槌を返すときは短く、内容に踏み込みません。

## 2. 設計の核（動きの検出と、音を出すを分ける）

オンラインでは、頷きの瞬間に毎回返す必要は薄いです。区切れを優先し、そこでまとめて返します。そのために、次の3つを分けます。

まず、IMUの動きを短い時間のまとまりとして切り出します。これをエピソード（episode）と呼びます。次に、話し手側の区切れなど「返してよいタイミング」だけで、貯めた情報を見て判断します。最後に、返すと決めた場合だけ、相槌の種類と文言を選びます。

## 3. データの形（エージェントに渡すもの）

エージェントに渡すのは、サインの有無ではなく、判断できる材料です。生データをそのまま全部投げるのではなく、動きを数値で短く要約したものを渡します。ここでは、この要約した数値を特徴量（feature）と呼びます。

渡す情報は、まず次の4ブロックにします。

- 会話の状況（今は区切れ直後か、音声は鳴っているか、直近の相槌頻度）
- 文字起こし（直近の流れと、過去の要点のサンプル）
- IMUの要約値（2秒、10秒、60秒、300秒など複数の窓）
- 直近の動きのまとまり（開始と終了、強さ、往復の回数、姿勢が変わったままか）

この形にすると、メモで下を向いた動きと、短い頷きの違いを材料として渡せます。

## 4. 判定のタイミング（区切れ優先の入口）

相槌は、返してよいタイミングでだけ判断します。まずは、TTS の小さなチャンク終わりをそのタイミングにします。将来的にマイク入力にする場合は、無音区間なども候補になります。

大事なのは、IMUの動きのまとまりを「すぐ鳴らす」のではなく、「次の区切れまで保持する」ことです。保持は固定秒で切り捨てるより、次の区切れで一回評価して、使わないなら捨てる方が扱いやすいです。

## 5. エージェントの構成（二段にして、理由を物語にしない）

エージェントは二段にします。

最初の段は「返すか返さないか」だけを決めます。ここで迷うなら返しません。次の段は「返す」と決めた場合だけ、相槌の種類と文言を選びます。

理由は自由作文にしない方が良いです。自己正当化が起きやすいからです。代わりに、根拠は短いチェック項目の集合にします。たとえば、区切れ直後、直近の相槌頻度が高い、IMUの往復が少ない、などのように、実際に渡した数値や判定に紐づく形にします。

## 6. パーソナライズ（正例と負例を取る）

弱い頷き、強い頷き、弱い首振り、強い首振りに加えて、負例も取ります。負例は、メモを取る動き、体勢を変える動き、視線を落とす動きです。

ここでやりたいことは、しきい値を厳しくするだけではなく、個人の動きの癖を「似ているか」で見られる材料を作ることです。似ているかの計算は単純でよく、まずは要約値の距離で十分です。エージェントには、その距離や近い例を渡して判断材料にします。

## 7. 評価のしかた（改善できるログ）

改善のためには、後から説明できるログが必要です。最低限、次を毎回残します。

- 判断点の時刻と、直前に読んだ文字起こし
- 直近の動きのまとまりの要約
- 返したか、返さなかったか
- 返した場合は、何を鳴らしたか
- 根拠のチェック項目（短いコード）

ここまで揃うと、ズレの原因が「タイミング」「検知」「返しすぎ」のどれかに分けやすくなります。

## 8. 実装の順番（最短で効く順）

まずは小さく効くところから入れます。

1. 相槌を出すタイミングを区切れに固定し、IMUは保持する
2. 二段判断を入れ、返さないを強くする
3. IMUの要約値を増やし、動きのまとまりを作って渡す
4. 負例を含むパーソナライズ計測を追加する
5. ログと表示を整えて、実験で回す

## 9. 触る場所（ファイルの目安）

この順で触ると、変更が追いやすいです。

- `app/transcript/timeline.py` と `app/transcript/speaker.py`: 文字起こしを短く区切る（判断点を増やす）
- `app/imu`: 動きのまとまりの抽出と保持（直近だけでなく流れを作る）
- `app/runtime/session.py`: 区切れでだけ判断し、必要なら再生が終わるまで読み上げを待つ
- `app/agents/backchannel_graph.py`: 二段判断と、根拠をチェック項目に寄せる
